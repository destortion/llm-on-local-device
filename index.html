I Built a Full LLM Chat App on My Phone — No Cloud, No GPU

Running Phi-3 Mini 3.8B fully on-device via llama.cpp on Android. Real benchmarks, OCR pipeline, slash commands — all in Termux.

✍ Satyam Sharma 📅 April 2026 ⏱ 8 min read

After building a real-time vision AI system with MobileNetV2, I had a new question: can I run an actual language model — something that can hold a conversation, summarize text, explain concepts — entirely on a phone with no internet?

The answer is yes. This is the story of how I went from zero to a fully working LLM chat app with streaming responses, OCR, slash commands, and real benchmark numbers — all running locally on a Samsung Galaxy S24 using Termux.

1 Building llama.cpp on Android

llama.cpp is a C++ inference engine that runs quantized LLMs on CPU. It compiles cleanly on Termux's aarch64 toolchain — no modifications needed.

# Install build tools
pkg install -y cmake ninja clang git python

# Build llama.cpp
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp
cmake -B build
cmake --build build -j4
Why llama.cpp? It runs on CPU-only with no external dependencies. The entire inference stack — tokenizer, attention, sampling — is self-contained. Perfect for Termux where you can't install CUDA or ROCm.

2 Choosing the Right Model

I started with TinyLlama 1.1B — fast, but gave random outputs for basic questions. Switched to Phi-3 Mini 3.8B from Microsoft, quantized to Q4_K_M. Night and day difference in quality.

3 Building the Chat UI

A Flask backend proxies requests to llama-server using the OpenAI-compatible API. The frontend streams tokens as they arrive via Server-Sent Events — so the response appears word by word, just like ChatGPT.

What is quantization in machine learning?
Phi-3 Mini
Quantization reduces a model's numerical precision — for example from 32-bit floats to 4-bit integers. This shrinks the model size and speeds up inference, with minimal impact on output quality. It's what makes running a 3.8B parameter model on a phone practical.

/summarize Quantization is a technique that maps high-precision values to lower-precision representations to reduce memory and compute requirements while preserving most of the original model accuracy.
Phi-3 Mini
Quantization converts high-precision model values to lower-precision ones, cutting memory and compute needs while keeping accuracy mostly intact.
Streaming via SSE: The Flask backend streams each token as it's generated using text/event-stream. The frontend reads the stream with a ReadableStream and appends tokens to the message in real time.

4 Slash Command System

I built a plugin-style slash command system. Type / and an autocomplete menu appears. Each command runs a specific pipeline — some hit the LLM, some run local tools.

/extractTake a photo or upload an image → extract text via Tesseract OCR
/deepsearchOCR the image, then feed extracted text to the LLM for deep analysis
/summarizeSummarize any text you paste after the command
/explainExplain something in simple terms
/bulletsConvert text into bullet points
/calcEvaluate math expressions — e.g. /calc sqrt(144)
/timeShow current date and time
/clearClear the chat history

Key insight: More threads ≠ faster. The Snapdragon 8 Gen 3 has 4 performance cores and 4 efficiency cores. Using all 8 pulls in the slow efficiency cores and causes thermal throttling. 4 threads on perf-cores only wins every time.

7 What I Learned

Model size isn't everything TinyLlama (1.1B) was nearly twice as fast but gave random, unusable outputs. Phi-3 Mini (3.8B) is slower but actually answers questions correctly. Quality matters more than speed for a real assistant.
Quantization is a superpower Q4_K_M brings a 7.6 GB model down to 2.23 GB — a 70% reduction — with barely noticeable quality loss. Without quantization, Phi-3 Mini simply wouldn't fit in phone RAM.
Thread tuning matters on mobile Blindly throwing all 8 cores at the problem gave worse performance than targeted 4-core inference due to thermal throttling and the efficiency/performance core split in Snapdragon.
The phone is more capable than you think 8.7 tokens/second means a 100-word reply in ~12 seconds. That's fast enough to be genuinely useful for summarization, explanation, and Q&A — entirely offline.

The full stack runs on any Android phone with Termux. No special hardware required.

# 5. Start chat UI (Flask)
cd llm-chat && python app.py
