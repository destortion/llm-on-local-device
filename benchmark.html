Why Phi-3 Mini?

Microsoft's Phi-3 Mini is a 3.8B parameter model that punches well above its size — designed specifically for edge devices. At Q4_K_M quantization it fits in ~2.3 GB of RAM, making it one of the few genuinely capable models that can run on a phone.

03 — Setup

How to Reproduce This

Everything runs inside Termux on Android — no root required. The full stack: llama.cpp for inference, Flask for the web UI, and a custom slash-command chat interface built from scratch.

# 1. Install Termux + build tools pkg install -y cmake ninja clang git python # 2. Build llama.cpp git clone https://github.com/ggerganov/llama.cpp cd llama.cpp && cmake -B build && cmake --build build -j4 # 3. Download Phi-3 Mini Q4 curl -L -o models/phi3-mini.gguf \ "https://huggingface.co/microsoft/Phi-3-mini-4k-instruct-gguf/resolve/main/Phi-3-mini-4k-instruct-q4.gguf" # 4. Start inference server ./build/bin/llama-server -m models/phi3-mini.gguf -c 4096 -t 4 --port 8080 # 5. Run the benchmark ./build/bin/llama-bench -m models/phi3-mini.gguf -o md -r 3 -t 4
04 — Analysis

What the Numbers Mean

At 8.7 tokens/second, a 100-token reply takes about 11 seconds. That's not fast, but it's usable — especially for offline tasks like document summarization, OCR analysis, or local Q&A.

For context, human reading speed is ~4–5 words/sec (~5–6 tokens/sec). So the model generates text just slightly faster than you can comfortably read it — which is the right target for a local assistant.

Key Takeaway

The bottleneck isn't the model — it's memory bandwidth. The Snapdragon 8 Gen 3 has enough CPU muscle, but moving 2.23 GB of weights through RAM repeatedly is the real cost. Quantization (Q4_K_M) gets that cost 70% cheaper than float16 with minimal quality loss.

The 4-thread sweet spot is because the Snapdragon has 4 performance cores and 4 efficiency cores. Using all 8 threads pulls in the efficiency cores, which are slower and cause load imbalance — so 4 threads on the perf cores only is faster.

05 — Application

What I Built On Top

The inference server is just the foundation. On top of it I built a full chat UI with:

  • Streaming responses via Server-Sent Events
  • Slash commands — /summarize, /explain, /bullets, /calc
  • OCR pipeline — snap a photo, extract text, analyze with AI in one command (/deepsearch)
  • Stop generation mid-stream
  • Dark/light theme, per-message copy, token stats

All running on the same phone. No API keys, no data leaving the device.
