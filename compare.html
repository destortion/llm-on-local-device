Gemma's prefill curve is inverted. Phi-3 Mini's throughput falls as prompts grow (34.63 → 23.88 t/s). Gemma 2 2B's throughput rises from pp32 to pp256 (68.77 → 71.62) before dropping at pp512. 03 — Time to First Token

TTFT: Wall-Clock Wait Time

The user-visible cost. TTFT = prompt_tokens ÷ prefill_speed. Gemma's 2.5× prefill advantage compounds into dramatically shorter waits on longer prompts.

Prompt lengthGemma TTFTPhi-3 TTFTGemma faster byUX (Gemma)
pp32 (~25 words)0.47 s0.92 s2.0×Instant
pp64 (~50 words)0.91 s1.88 s2.1×Instant
pp128 (~100 words)1.79 s4.31 s2.4×Instant
pp256 (~200 words)3.57 s7.83 s2.2×Acceptable
pp512 (~400 words)8.54 s21.4 s2.5×Noticeable
The 8-second UX boundary shifts dramatically. Phi-3 crosses the 8-second TTFT threshold at ~230 tokens. Gemma crosses it at ~512 tokens — meaning Gemma can handle prompts more than twice as long before hitting the "this feels slow" wall. For document analysis, OCR pipelines, and long system prompts, this is a significant practical difference.

Gemma unlocks a new config option. For generation, both models confirm: 4 threads beats 8 by ~21%. But for Gemma's prefill, 8 threads is actually faster (72.41 vs 62.21 t/s). Gemma's smaller weight matrices give the efficiency cores enough useful work to do without creating synchronization bottlenecks. This means for long-prompt, short-response workloads (document summarization, OCR analysis) you could switch to 8 threads and get faster TTFT with acceptable generation speed.
05 — Memory Bandwidth

Effective DRAM Bandwidth

Generation speed × model size = bandwidth consumed per decode pass. A higher number means more efficient use of available memory bandwidth.

Gemma bandwidth = 14.16 t/s × 1.59 GB = 22.5 GB/s (29% of LPDDR5X 77 GB/s)
Phi-3 bandwidth = 8.70 t/s × 2.23 GB = 19.4 GB/s (25% of LPDDR5X 77 GB/s)

Gemma achieves higher effective bandwidth utilization (29% vs 25%) — its smaller weight matrices fit more efficiently into the CPU's memory access pipeline, resulting in better bus utilization per thread. Both models are well below the 77 GB/s LPDDR5X ceiling; the bottleneck remains the CPU's ability to issue load instructions, not DRAM capacity.
